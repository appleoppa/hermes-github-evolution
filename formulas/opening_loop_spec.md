# 开智进化循环执行规范摘要

> 版本：V2.0-current
> 状态：现行唯一执行规范

## 完整公式

**公式审错 → 21354×5 + 12534×5 + 14325×5 → 对标找短板 → EVM缺陷治理 → 超级路由 → GitHub工厂外部学习 → 河图洛书反证仲裁 → 补齐短板 → 进化基因入库**

## 三重门禁

| 门禁 | 必须输出 |
|---|---|
| **EVM缺陷治理门** | 短板原文、缺陷类别、治理前数值、治理后数值、缺陷压降、剩余风险；门禁阈值：缺陷率≥0.75通过，<0.6阻塞 |
| **超级路由决策门** | 主学习路径、替代路径、是否调用GitHub工厂、是否调用多模型反证、最小补齐动作候选、失败修复路径 |
| **GitHub工厂学习门** | 实际来源（仓库+链接+star）、原文具体机制（≥50字）、与短板对应关系、学习点改变补齐动作；参与等级：落地/判断/信息/通路/未参与 |
| **河图洛书反证仲裁门** | 主脑统筹×反证审错×修复落地×旁证压缩×主脑收束；参与等级：落地参与/判断参与/本地推演/未参与 |

## 核心原则

- 真实开智不以产物存在为完成证明
- 评估器必须先审自己
- 自我反思必须绑定下一步行为变化
- 自动化流水线必须有失败条件
- 报告存在 ≠ 循环完成
