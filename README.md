# GitHub 进化闭环系统

## 定位

本目录用于承载 Hermes 的 GitHub 进化闭环：本地只做调度、验收和吸收，GitHub 负责远程检索、容器执行、自动化验证和成果回流。

## 主辅脑配置

- 主要调度：MiniMax，负责任务分解、节奏控制、资源分配和检查点组织。
- 最强主脑：GPT-5.5，负责最终裁决、核心推理、APEX/EVM收束；进化以GPT为主。
- 辅助脑：DeepSeek，负责反证审错、漏洞发现、风险提示和补充推理。

三类模型密钥均已在本机环境中存在，配置文件只保留环境变量引用，不写明文密钥。

默认决策顺序：MiniMax先调度 → GPT主脑判断 → DeepSeek反证辅助 → GPT最终收束。

## GitHub 闭环结构

- tasks：待执行任务与 Gist 中转任务
- research：顶级项目、论文、架构精华提炼
- genes：进化基因沉淀
- experiments：代码实验与验证
- actions：远程自动化流水线设计
- inbox：GitHub 回流到本地的成果队列
- archive：历史沉淀

## 最小运行链路

1. 本地 Hermes 生成任务包
2. 写入 GitHub 仓库或 Gist
3. GitHub Actions 在容器中执行检索、提炼、验证
4. 结果写回仓库
5. 本地 Hermes 拉取成果，进行真实性核验
6. 合格成果进入进化基因库或技能库

## 安全边界

- 令牌只放在环境变量或 GitHub Secrets
- 不在报告、记忆、技能或日志中输出令牌明文
- 仓库自动化优先用细粒度权限
- 删除和覆盖动作必须先读取实际内容再执行
