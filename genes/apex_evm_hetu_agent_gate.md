# APEX + EVM + GitHub + 河图洛书最强智能体门禁

本门禁来自一次连续真实开智周期。

## 核心判断

打造最强智能体不是堆更多工具，也不是生成更多周期产物，而是每一轮都必须穿过四道门：

1. APEX：评分器先审自己，禁止手写修复后分数，禁止循环论证。
2. EVM：缺陷压降必须来自真实修复动作，禁止重复使用同一治理系数自证提升。
3. GitHub：外部学习必须读取原文，并把学习点转成下一入口门禁。
4. 河图洛书：五段判断链必须包含反证、仲裁、落地和收束，不能只是角色名称排列。

## 连续执行停止条件

任一周期出现以下情况，必须停止并修复，不得继续冒充进化：

- 没有外部原文阅读；
- 没有总公式代入；
- 没有EVM缺陷映射和压降；
- 没有河图洛书五段链；
- 学习点没有改变下一轮入口；
- 多回合学习点高度重复，说明进入模板化。

## 下次入口

先检查上一周期形成的门禁是否实际生效，再开始新周期。
